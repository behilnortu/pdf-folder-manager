## Why Topics Matter:

- **Discoverability**: People can find your project when searching for related topics
- **Categorization**: GitHub uses topics to categorize and recommend projects
- **SEO**: Improves search engine visibility
- **Community**: Helps connect your project with similar projects

## Best Practices:

- Use 5-15 topics (most relevant ones)
- Use lowercase
- Use hyphens for multi-word topics (e.g., `pdf-viewer` not `pdf viewer`)
- Include technology stack (nodejs, express, javascript)
- Include functionality (search, upload, real-time)
- Include use cases (document-management, file-manager)
